üî∫ Hill Climbing

Hill Climbing (escalada de colina) es un algoritmo de b√∫squeda heur√≠stica utilizado para resolver problemas de optimizaci√≥n. Su objetivo es encontrar la mejor soluci√≥n posible partiendo de una soluci√≥n inicial y mejor√°ndola gradualmente mediante peque√±os cambios. El algoritmo explora el entorno inmediato de la soluci√≥n actual y se mueve hacia la direcci√≥n donde mejora el valor de la funci√≥n objetivo.

  • Proceso iterativo: en cada paso, se examinan las soluciones vecinas y se selecciona la mejor de ellas.
  • √ìptimo local: el algoritmo puede detenerse en una soluci√≥n que es la mejor en su vecindad, pero no necesariamente la mejor globalmente.
  • Aplicaci√≥n: es √∫til para problemas donde se puede definir una funci√≥n de evaluaci√≥n y generar soluciones cercanas, como problemas de planificaci√≥n o asignaci√≥n.
  • Limitaci√≥n: no explora todo el espacio de soluciones, por lo que no garantiza encontrar la soluci√≥n √≥ptima global.

üß¨ Algoritmo Gen√©tico

Un Algoritmo Gen√©tico (GA) es una t√©cnica de optimizaci√≥n basada en los principios de la evoluci√≥n natural, como la selecci√≥n, cruza (crossover) y mutaci√≥n. Funciona con una poblaci√≥n de posibles soluciones, representadas como cromosomas, que evolucionan a lo largo de varias generaciones con el objetivo de encontrar una soluci√≥n √≥ptima o cercana al √≥ptimo.

  • Representaci√≥n gen√©tica: cada soluci√≥n se codifica como una cadena (por ejemplo, un vector binario o num√©rico).
  • Evaluaci√≥n por fitness: cada individuo es evaluado usando una funci√≥n de aptitud (fitness function) que mide qu√© tan buena es la soluci√≥n.
  • Cruce y mutaci√≥n: se combinan soluciones para crear nuevas generaciones y se introducen peque√±as alteraciones para mantener la diversidad.
  • Exploraci√≥n global: al trabajar con muchas soluciones a la vez, tiene mayor capacidad para evitar quedarse atrapado en √≥ptimos locales.
  • Uso en problemas complejos: como dise√±o de redes, programaci√≥n gen√©tica, optimizaci√≥n combinatoria, y aprendizaje autom√°tico.

Enunciado 2: Optimizaci√≥n del Horario de Estudio

Un estudiante tiene disponibles 10 franjas horarias libres durante la semana, cada una con una duraci√≥n y un nivel estimado de productividad (del 1 al 10). El objetivo es seleccionar las mejores franjas para maximizar su productividad semanal total, pero con una restricci√≥n: no debe estudiar m√°s de 15 horas en total. Implementa un algoritmo de hill climbing que, partiendo de una selecci√≥n aleatoria de franjas horarias, busque mejorar la productividad total sin superar el l√≠mite de horas. Usa Pandas para representar las franjas y NumPy para mutar la soluci√≥n. 


import numpy as np
import pandas as pd

data = {
    'Franja': ['L1', 'L2', 'M1', 'M2', 'M3', 'J1', 'J2', 'V1', 'V2', 'S1'],
    'Duraci√≥n': [2, 1.5, 2, 1, 3, 2.5, 1.5, 2, 3, 1],
    'Productividad': [8, 6, 7, 5, 9, 8, 7, 6, 5, 4]
}
franjas = pd.DataFrame(data)

def evaluar_solucion(solucion, franjas, max_horas=15):
    horas_totales = np.sum(franjas['Duraci√≥n'][solucion])
    if horas_totales > max_horas:
        return -np.inf
    return np.sum(franjas['Productividad'][solucion])

def generar_solucion_inicial(n_franjas, n_seleccionar):
    return np.random.choice(n_franjas, size=n_seleccionar, replace=False)

def generar_vecino(solucion, n_franjas):
    vecino = solucion.copy()
    idx_cambiar = np.random.randint(len(vecino))
    posibles = [f for f in range(n_franjas) if f not in vecino]
    if not posibles:
        return vecino
    vecino[idx_cambiar] = np.random.choice(posibles)
    return vecino

def hill_climbing(franjas, max_horas=15, max_iter=1000):
    n_franjas = len(franjas)
    n_seleccionar = max(1, n_franjas // 2)
    solucion_actual = generar_solucion_inicial(n_franjas, n_seleccionar)
    valor_actual = evaluar_solucion(solucion_actual, franjas, max_horas)
    
    for _ in range(max_iter):
        vecino = generar_vecino(solucion_actual, n_franjas)
        valor_vecino = evaluar_solucion(vecino, franjas, max_horas)
        if valor_vecino > valor_actual:
            solucion_actual = vecino
            valor_actual = valor_vecino
    
    return solucion_actual, valor_actual

mejor_solucion, mejor_valor = hill_climbing(franjas)
print("Franjas:", franjas['Franja'][mejor_solucion].values)
print("Productividad:", mejor_valor)
print("Horas:", np.sum(franjas['Duraci√≥n'][mejor_solucion]))

Enunciado 3 (Selecci√≥n √≥ptima de platos en el men√∫ universitario):

En la cafeter√≠a universitaria hay 8 platos diferentes. Cada plato tiene un precio, una cantidad de calor√≠as y una puntuaci√≥n de satisfacci√≥n (seg√∫n encuestas de estudiantes). Un estudiante solo tiene S/20 para gastar por d√≠a y quiere obtener el m√°ximo nivel de satisfacci√≥n sin pasarse del presupuesto ni consumir m√°s de 1000 calor√≠as. Utiliza Pandas para representar el men√∫ y NumPy para generar combinaciones de platos. Aplica hill climbing para encontrar la mejor combinaci√≥n de 3 platos que maximicen la satisfacci√≥n bajo las restricciones indicadas. 


          import numpy as np
          import pandas as pd
          
          data = {
              'Plato': ['P1', 'P2', 'P3', 'P4', 'P5', 'P6', 'P7', 'P8'],
              'Precio': [5, 7, 4, 8, 6, 9, 5, 7],

              'Satisfacci√≥n': [7, 8, 6, 9, 7, 8, 5, 6]
          }
          menu = pd.DataFrame(data)
          
          def evaluar_solucion(solucion, menu, max_precio=20, max_calorias=1000):
              precio_total = np.sum(menu['Precio'][solucion])
              calorias_total = np.sum(menu['Calor√≠as'][solucion])
              if precio_total > max_precio or calorias_total > max_calorias:
                  return -np.inf
              return np.sum(menu['Satisfacci√≥n'][solucion])
          
          def generar_solucion_inicial(n_platos, n_seleccionar=3):
              return np.random.choice(n_platos, size=n_seleccionar, replace=False)
          
          def generar_vecino(solucion, n_platos):
              vecino = solucion.copy()
              idx_cambiar = np.random.randint(len(vecino))
              posibles = [p for p in range(n_platos) if p not in vecino]
              if not posibles:
                  return vecino
              vecino[idx_cambiar] = np.random.choice(posibles)
              return vecino
          
          def hill_climbing(menu, max_precio=20, max_calorias=1000, max_iter=1000):
              n_platos = len(menu)
              solucion_actual = generar_solucion_inicial(n_platos)
              valor_actual = evaluar_solucion(solucion_actual, menu, max_precio, max_calorias)
              
              for _ in range(max_iter):
                  vecino = generar_vecino(solucion_actual, n_platos)
                  valor_vecino = evaluar_solucion(vecino, menu, max_precio, max_calorias)
                  if valor_vecino > valor_actual:
                      solucion_actual = vecino
                      valor_actual = valor_vecino
              
              return solucion_actual, valor_actual
          
          mejor_solucion, mejor_valor = hill_climbing(menu)
          print("Platos:", menu['Plato'][mejor_solucion].values)
          print("Satisfacci√≥n:", mejor_valor)
          print("Precio:", np.sum(menu['Precio'][mejor_solucion]))
          print("Calor√≠as:", np.sum(menu['Calor√≠as'][mejor_solucion]))
